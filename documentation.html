Presentation du projet

YouTube Downloader Node est une application web locale qui permet de telecharger des videos et de l'audio depuis YouTube. Elle tourne entierement sur votre ordinateur (rien n'est envoye sur internet sauf les requetes vers YouTube).

Ce que fait le programme

  • Telecharger des videos YouTube en MP4, MKV ou WEBM
  • Extraire l'audio en MP3, FLAC, WAV, AAC ou OGG
  • Choisir la qualite (1080p, 720p, etc. pour la video / haute, moyenne, basse pour l'audio)
  • Telecharger des playlists entieres
  • Rechercher des videos YouTube directement depuis l'application
  • Organiser les fichiers telecharges dans une bibliotheque locale avec des dossiers
  • Gerer une file d'attente de telechargements
  • Recuperer la couverture/miniature des videos
  • Utiliser une extension de navigateur pour telecharger directement depuis YouTube

Les technologies utilisees

Technologie Role Explication simple
Node.js Moteur du serveur Permet d'executer du JavaScript en dehors du navigateur, c'est le "cerveau" de l'application
Express Serveur web Bibliotheque qui cree un serveur web et gere les requetes HTTP (les URL que vous visitez)
Python Langage requis Necessaire car yt-dlp est ecrit en Python
yt-dlp Moteur de telechargement Programme Python qui sait comment telecharger depuis YouTube (gere les formats, DRM, etc.)
ffmpeg Conversion audio/video Outil qui convertit les fichiers entre formats (ex: convertir en MP3) et fusionne audio+video

Sur un Windows tout neuf (sans aucun programme installe), voici ce dont vous avez besoin. Le script start.bat installe tout automatiquement, mais voici le detail :

Installation automatique (recommandee)

Le fichier start.bat fait tout automatiquement. Il suffit de double-cliquer dessus. Voici exactement ce qu'il fait, etape par etape :

1

Verification de Node.js

Le script cherche si node est installe sur votre PC. Si non, il l'installe via :

winget install OpenJS.NodeJS.LTS

Apres l'installation, il vous demande de fermer et relancer le script (car le PATH systeme doit etre mis a jour pour reconnaitre la commande node).

2

Verification de Python

Meme chose pour Python. Si absent :

winget install Python.Python.3.12

Relancer le script apres installation.

3

Installation de yt-dlp

Verifie si le module Python yt_dlp est disponible. Si non :

python -m pip install yt-dlp

pip est le gestionnaire de paquets de Python (installe avec Python). Pas besoin de relancer ici.

4

Verification de ffmpeg

Si ffmpeg n'est pas trouve :

winget install Gyan.FFmpeg

Relancer le script apres installation.

5

Installation des dependances Node.js

Si le dossier node_modules n'existe pas, il lance :

npm install

Cela lit le fichier package.json et telecharge les deux bibliotheques necessaires :

  • express - pour creer le serveur web
  • cors - pour autoriser les requetes depuis l'extension navigateur
6

Creation des dossiers et fichiers

Le script cree les dossiers et fichiers de donnees s'ils n'existent pas :

downloads/          ← ou seront stockes les fichiers telecharges
data/               ← ou sont stockees les donnees de l'application
data/library.json   ← {"folders":[],"items":[]}
data/profiles.json  ← []
data/history.json   ← []
data/queue.json     ← []
data/notifications.json ← []
7

Lancement

Le script ouvre votre navigateur par defaut a l'adresse http://localhost:3000 puis lance le serveur Node.js :

start http://localhost:3000
node server.js
Important : les relances La premiere fois, vous devrez peut-etre lancer start.bat jusqu'a 4 fois car chaque installation de programme (Node.js, Python, ffmpeg) necessite un redemarrage du script pour que Windows reconnaisse la nouvelle commande.

Installation manuelle

Si vous preferez tout faire a la main ou si start.bat ne fonctionne pas :

1. Installer Node.js

Telechargez le depuis https://nodejs.org (version LTS). Lancez l'installeur et suivez les etapes.

Verifiez dans un terminal :

node --version    ← doit afficher v20.x.x ou plus
npm --version     ← doit afficher 10.x.x ou plus

2. Installer Python

Telechargez le depuis https://python.org (version 3.12+).

Attention ! Lors de l'installation, cochez absolument la case "Add Python to PATH" en bas de la premiere fenetre. Sans ca, Windows ne trouvera pas la commande python. 
python --version  ← doit afficher Python 3.12.x

3. Installer yt-dlp

python -m pip install yt-dlp

Verification :

python -m yt_dlp --version

4. Installer ffmpeg

Via winget :

winget install Gyan.FFmpeg

Ou telechargez manuellement depuis https://ffmpeg.org/download.html et ajoutez le dossier bin au PATH systeme.

ffmpeg -version   ← doit afficher la version

5. Installer les dependances du projet

Ouvrez un terminal dans le dossier du projet :

cd chemin/vers/youtube_dowloader_node
npm install

6. Creer les dossiers

mkdir downloads
mkdir data

Le serveur (server.js)

C'est le point d'entree du programme. Quand vous lancez node server.js, voici ce qui se passe dans l'ordre :

Etape 1 : Chargement des modules

const express = require('express');   // Serveur web
const cors = require('cors');         // Autorise les requetes cross-origin
const path = require('path');         // Gestion des chemins de fichiers
const fs = require('fs');             // Lecture/ecriture de fichiers
const config = require('./config');   // Configuration auto-detectee

require() est la facon dont Node.js importe des modules. C'est comme dire "j'ai besoin de cet outil, charge-le en memoire".

Etape 2 : Configuration des middlewares

const app = express();            // Cree une application Express
app.use(cors());                  // Autorise les requetes depuis d'autres origines
app.use(express.json());          // Comprend les donnees JSON dans les requetes
app.use(express.urlencoded({ extended: true })); // Comprend les formulaires

Un middleware est un traitement que chaque requete subit avant d'arriver a sa destination. C'est comme un controle de securite a l'entree d'un batiment.

Etape 3 : Fichiers statiques

app.use(express.static(path.join(__dirname, 'public')));
app.use('/downloads', express.static(config.DOWNLOADS_DIR));

Le serveur sert directement les fichiers du dossier public/ (HTML, CSS, JS) et rend les fichiers telecharges accessibles via l'URL /downloads/nom-du-fichier.mp3.

Etape 4 : Montage des routes API

app.use('/api/download', require('./api/download'));
app.use('/api/info', require('./api/info'));
app.use('/api/progress', require('./api/progress'));
// ... 8 autres routes

Chaque app.use('/api/xxx', ...) dit a Express : "quand quelqu'un visite /api/xxx, utilise le code dans le fichier api/xxx.js".

Etape 5 : Demarrage

app.listen(config.PORT, () => {
    console.log(`YouTube Downloader demarre sur http://localhost:${config.PORT}`);
});

Le serveur commence a ecouter sur le port 3000. Il est pret a recevoir des requetes.

La configuration (config.js)

Ce fichier detecte automatiquement ou se trouvent les programmes necessaires sur votre PC. Aucune configuration manuelle n'est necessaire.

Ce qu'il detecte :

1

Python

Cherche python ou python3 dans le PATH systeme avec la commande where.

2

ffmpeg

Cherche ffmpeg dans le PATH et recupere le chemin de son dossier parent.

3

yt-dlp

Si Python est trouve, utilise python -m yt_dlp (plus fiable sur Windows car evite le blocage "Smart App Control"). Sinon, tente yt-dlp directement.

Les valeurs de configuration :

{
    PORT: 3000,                              // Port du serveur
    DOWNLOADS_DIR: "./downloads",            // Dossier des telechargements
    DATA_DIR: "./data",                      // Dossier des donnees

    AUDIO_FORMATS: ['mp3', 'flac', 'wav', 'aac', 'ogg'],
    VIDEO_FORMATS: ['mp4', 'mkv', 'webm'],
    AUDIO_QUALITIES: ['0', '5', '9'],        // 0 = meilleure, 9 = plus legere
    VIDEO_QUALITIES: ['best', '1080', '720', '480', '360'],

    // Validation des URLs YouTube et des IDs de jobs
    YOUTUBE_URL_PATTERN: /^https?:\/\/(www\.)?(youtube\.com|youtu\.be)\/...$/,
    JOB_ID_PATTERN: /^yt_[a-f0-9]+$/
}

Le worker (worker.js)

C'est le coeur du telechargement. Ce fichier est lance comme un processus independant pour chaque telechargement. Cela permet au serveur de continuer a repondre pendant que le telechargement tourne en arriere-plan.

Comment il est lance :

// Dans api/download.js, le serveur fait :
spawn('node', ['worker.js', jobId, url, cover, type, format, quality]);

Le worker recoit 6 parametres :

ParametreExempleDescription
jobIdyt_a1b2c3d4Identifiant unique du telechargement
urlhttps://youtube.com/watch?v=xxxURL de la video
cover11 = sauver la miniature, 0 = non
typeaudioaudio ou video
formatmp3Format de sortie
quality0Qualite (0=meilleure pour audio, 1080=Full HD pour video)

Le flux complet du worker :

1. Recoit les parametres (jobId, url, format, qualite...)
   |
2. Cree un fichier de log : downloads/{jobId}.log
   |     ← "Demarrage..."
   |
3. Construit la commande yt-dlp adaptee au type demande
   |     Ex pour audio : python -m yt_dlp -x --audio-format mp3 ...
   |     Ex pour video : python -m yt_dlp -f "bestvideo+bestaudio" ...
   |
4. Execute la commande yt-dlp via spawn()
   |     ← Ecrit la progression dans le fichier .log en temps reel
   |     ← Le frontend lit ce .log pour afficher la barre de progression
   |
5. yt-dlp termine
   |
6. Cherche le fichier telecharge (downloads/{jobId}.mp3)
   |     ← Gere les noms de fichiers avec suffixes (ex: .f399.mp4)
   |
7. Gere la couverture (si demandee)
   |     a. Cherche une image ecrite par yt-dlp (thumbnail)
   |     b. Si pas trouvee : telecharge depuis i.ytimg.com
   |     c. Convertit en JPG si necessaire via ffmpeg
   |
8. Renomme le fichier avec le titre de la video YouTube
   |     Ex: yt_a1b2c3d4.mp3 → Ma Super Chanson.mp3
   |
9. Nettoie les fichiers temporaires
   |
10. Cree le fichier .done avec les infos finales
       downloads/{jobId}.done = {"file":"Ma Super Chanson.mp3","cover":"Ma Super Chanson.jpg"}

Exemple de commande yt-dlp generee :

Pour telecharger de l'audio MP3 en qualite maximale :

python -m yt_dlp \
  --ffmpeg-location "C:/Program Files/ffmpeg/bin" \
  --newline --progress-delta 1 \
  -o "downloads/yt_a1b2c3d4.%(ext)s" \
  --no-playlist \
  -x --audio-format mp3 --audio-quality 0 \
  --embed-thumbnail --write-thumbnail --convert-thumbnails jpg \
  "https://youtube.com/watch?v=dQw4w9WgXcQ"

Pour telecharger une video MP4 en 1080p :

python -m yt_dlp \
  --ffmpeg-location "C:/Program Files/ffmpeg/bin" \
  --newline --progress-delta 1 \
  -o "downloads/yt_a1b2c3d4.%(ext)s" \
  --no-playlist \
  -f "bestvideo[height<=1080]+bestaudio/best[height<=1080]" \
  --merge-output-format mp4 \
  --postprocessor-args "Merger+ffmpeg_o:-c:v copy -c:a aac" \
  --embed-thumbnail --convert-thumbnails jpg \
  "https://youtube.com/watch?v=dQw4w9WgXcQ"

Les routes API (api/*.js)

Les routes API sont les "portes d'entree" du serveur. Chaque fichier dans le dossier api/ gere un type de fonctionnalite. Le frontend (la page web) communique avec le serveur en envoyant des requetes HTTP a ces routes.

C'est quoi une API ? Une API (Application Programming Interface) est un ensemble de "commandes" que le frontend peut envoyer au serveur. Par exemple, quand vous cliquez sur "Telecharger", le frontend envoie une requete POST /api/download avec l'URL de la video. Le serveur repond avec un identifiant de job pour suivre la progression.

Flux de telechargement pas a pas

Voici exactement ce qui se passe quand vous collez une URL YouTube et cliquez sur "Telecharger" :

1

L'utilisateur colle une URL

Le frontend (app.js) detecte l'URL et envoie une requete au serveur pour recuperer les infos :

POST /api/info
Body: { "url": "https://youtube.com/watch?v=..." }

Le serveur execute yt-dlp --dump-json et retourne le titre, la miniature, la duree, etc.

2

L'utilisateur clique sur "Telecharger"

Le frontend envoie :

POST /api/download
Body: { "url": "...", "type": "audio", "format": "mp3", "quality": "0", "cover": "1" }
3

Le serveur cree un job

Le route api/download.js :

  • Valide l'URL YouTube (verifie le format avec une regex)
  • Genere un identifiant unique : yt_ + hash hexadecimal aleatoire
  • Sanitize les parametres (verifie que le format et la qualite sont valides)
  • Lance le worker en arriere-plan : spawn('node', ['worker.js', ...])
  • Repond immediatement : { "success": true, "jobId": "yt_a1b2c3d4" }
4

Le frontend surveille la progression

Toutes les secondes, le frontend demande :

GET /api/progress?id=yt_a1b2c3d4

Le route api/progress.js lit le fichier downloads/yt_a1b2c3d4.log et en extrait le pourcentage de progression. Reponse :

{
    "status": "progress",
    "percent": 45,
    "message": "Telechargement 45%..."
}
5

Le telechargement se termine

Le worker cree un fichier yt_a1b2c3d4.done. Au prochain sondage, api/progress.js detecte ce fichier et repond :

{
    "status": "done",
    "percent": 100,
    "file": "downloads/Ma Super Chanson.mp3",
    "cover": "downloads/Ma Super Chanson.jpg"
}
6

Le fichier est disponible

Le frontend affiche un lien de telechargement. Le fichier est accessible via :

http://localhost:3000/downloads/Ma Super Chanson.mp3

Le fichier est aussi ajoute a la bibliotheque locale et a l'historique.

Navigateur → POST /api/download → spawn worker.js → yt-dlp

Navigateur ← GET /api/progress ← Lit fichier .log ← Progression

Navigateur ← GET /downloads/... ← Fichier final .mp3/.mp4

Extension navigateur

Le dossier extension/ contient une extension compatible Chrome, Edge et Brave. Elle ajoute des boutons de telechargement directement sur les pages YouTube.

Comment l'installer :

  1. Ouvrez votre navigateur (Chrome/Edge/Brave)
  2. Allez dans les parametres d'extensions :
    • Chrome : chrome://extensions
    • Edge : edge://extensions
  3. Activez le "Mode developpeur" (bouton en haut a droite)
  4. Cliquez sur "Charger l'extension non empaquetee"
  5. Selectionnez le dossier extension/ du projet

Ce que fait l'extension :

  • Ajoute un bouton de telechargement flottant sur les pages YouTube
  • Ajoute un bouton inline a cote des likes/dislikes
  • Panneau d'options pour choisir format/qualite sans quitter YouTube
  • Suivi de la progression directement sur la page YouTube
  • Fonctionne sur les videos, les shorts et les playlists
Prerequis de l'extension Le serveur (node server.js) doit etre lance pour que l'extension fonctionne. L'extension envoie les requetes a http://localhost:3000.

Depannage

Le programme ne se lance pas

Erreur : 'node' n'est pas reconnu Node.js n'est pas installe ou pas dans le PATH. Relancez start.bat ou installez manuellement.
Erreur : 'python' n'est pas reconnu Python n'est pas installe. Relancez start.bat ou installez depuis python.org (n'oubliez pas de cocher "Add to PATH").
Erreur : Cannot find module 'express' Les dependances ne sont pas installees. Lancez npm install dans le dossier du projet.

Le telechargement echoue

Erreur : ffmpeg non trouve Installez ffmpeg via winget install Gyan.FFmpeg puis fermez et rouvrez le terminal.
Erreur : "Video unavailable" ou "Sign in to confirm" Certaines videos sont protegees par YouTube. yt-dlp ne peut pas toujours les telecharger. Essayez de mettre a jour yt-dlp via l'interface (onglet systeme) ou manuellement : python -m pip install -U yt-dlp
Le telechargement reste bloque a 0% Verifiez que yt-dlp est installe : python -m yt_dlp --version. Si ca ne fonctionne pas, reinstallez : python -m pip install yt-dlp.

L'extension ne fonctionne pas

Le serveur doit tourner L'extension communique avec http://localhost:3000. Assurez-vous que le serveur est lance (node server.js ou start.bat) avant d'utiliser l'extension.

Commandes utiles pour diagnostiquer

# Verifier que tout est installe :
node --version          ← doit afficher v20+
python --version        ← doit afficher 3.12+
python -m yt_dlp --version   ← doit afficher 2024.x.x+
ffmpeg -version         ← doit afficher la version

# Mettre a jour yt-dlp :
python -m pip install -U yt-dlp

# Reinstaller les dependances Node.js :
rm -rf node_modules
npm install
