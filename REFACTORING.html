Helper 1 escapeHtml(s)

Avant defini 7 fois

    c => ({'<':'&lt;','>':'&gt;','&':'&amp;','"':'&quot;'}[c]));

// Dans showHourDetails

// Dans renderArtistChart
const esc = ... EXACTEMENT le meme code

// + 4 autres copies dans le fichier
Apres 1 helper global
// helpers.js, defini une seule fois
const escapeHtml = (s) =>
    String(s ?? '').replace(/[<>&"]/g,
        c => ({'<':'&lt;','>':'&gt;',
              '&':'&amp;','"':'&quot;'}[c]));

// Tous les call sites:
escapeHtml(title)
escapeHtml(artist)
// 7 endroits, 1 seule definition
Gain : ~30 lignes economisees + bug-proof (la prochaine modif corrige 7 endroits d'un coup). Bonus : ?? '' evite l'erreur sur null/undefined.

Helper 2 apiCall(url, opts)

Avant 119 fetch bruts
try {
    const resp = await fetch('api/flow?action=list');
    const data = await resp.json();
    if (!data.success) return;
    // pas de check Content-Type
    // pas de detection 404 = HTML
    flowTracks = data.tracks || [];
} catch (e) {
    // gestion d'erreur differente partout
    console.error('flow error', e);
}
// Pattern repete 119 fois
Apres 1 wrapper unifie
// helpers.js
async function apiCall(url, opts = {}) {
    const resp = await fetch(url, opts);
    const ct = resp.headers.get('content-type') || '';
    if (!ct.includes('application/json'))
        throw new Error('Serveur a redemarrer');
    return resp.json();
}

// Tous les call sites:
const data = await apiCall('api/flow?action=list');
Gain : ~150 lignes + detection auto du "serveur pas redemarre" (probleme recurrent dans nos echanges) + erreur unifiee.
